

Hermes Agent 项目针对**记忆系统、技能自主学习和自主进化**三大核心能力，制定了明确的**官方规则标准文档**。这些文档并非第三方解读，而是项目源码和官方文档中明确定义的技术规范，确保系统行为可预测、可扩展。以下是关键规则文档的总结与说明：

---

## 一、记忆系统规则文档
### 1. **`MEMORY.md` 与 `USER.md` 规范**  
- **作用**：定义**持久化记忆的存储格式与内容边界**，避免信息过载。  
- **核心规则**：  
  - **内容限制**：  
    - `MEMORY.md` 仅存储**环境事实与项目约定**（如技术栈、工具链配置），**严格限制在约 2,200 字符内**（约 800 tokens），确保系统提示词轻量高效。  
    - `USER.md` 仅记录**用户偏好与沟通风格**（如“回答需简洁”“代码用制表符”），上限约 1,375 字符。  
  - **禁止项**：  
    - 任务进度、临时状态、会话结果等**动态数据不得写入**，仅保留**稳定、高频复用的长期事实**。  
    - 敏感信息（如凭证、后门提示）会被**自动过滤**，防止泄露风险。  
- **操作规范**：  
  - 通过 `memory` 工具管理条目，支持 `add`/`replace`/`remove` 三类操作，**无需内部 ID，仅需匹配唯一子字符串**即可更新内容。  

### 2. **FTS5 会话检索标准**  
- **作用**：规范跨会话历史的**按需召回机制**，避免全量注入上下文。  
- **核心规则**：  
  - 所有会话历史存储于 **SQLite 数据库**（`~/.hermes/state.db`），通过 **FTS5 全文索引**实现毫秒级检索。  
  - 检索流程强制包含 **LLM 摘要步骤**：仅返回与当前任务相关的**压缩后上下文**，而非原始对话记录。  
  - **触发条件**：仅当用户明确提问“上周讨论过什么？”等情景性问题时激活，**非默认加载**。  

---

## 二、技能自主学习规则文档
### 1. **`agentskills.io` 开放标准**  
- **作用**：定义**Skill 文件的结构化格式**，确保跨 Agent 兼容性（如 OpenClaw、Claude Code 可复用）。  
- **核心规则**：  
  - **文件位置**：所有自动生成的 Skill 存储在 `~/.hermes/skills/` 目录，**必须为 Markdown 格式**。  
  - **必含字段**：  
    - **执行步骤**（按工具调用顺序的原子操作）。  
    - **关键决策点**（如“若文件存在则跳过创建”）。  
    - **常见失败模式**（如“依赖缺失时自动安装”）。  
    - **验证逻辑**（输出结果的自动检查方法）。  
  - **动态更新机制**：  
    - 仅当新路径**显著优于旧版本**（如步骤减少 30%）时触发更新。  
    - 采用 **Patch 式修改**（仅替换问题片段），**避免全量覆写导致逻辑丢失**。  

### 2. **技能生成与淘汰规则**  
- **触发条件**：任务需满足 **“足够新颖且非平凡”**，例如：  
  - 工具调用 **≥5 次**。  
  - 经历 **出错并自我修复**。  
  - 用户**显式确认**（如“好”）或**隐式采纳**（未修改直接使用）。  
- **淘汰机制**：  
  - 连续 **N 次未被匹配到的技能自动降权归档**，防止技能库膨胀。  
  - 过时技能**不会直接删除**，而是标记为历史版本供追溯。  

---

## 三、自主进化规则文档
### 1. **GEPA（Genetic Evolution of Prompt Architectures）算法规范**  
- **作用**：定义**Prompt 与技能的自动化进化流程**，通过遗传算法优化行为。  
- **核心规则**：  
  - **轨迹筛选**：仅保留 **评分 ≥7 分**（1-10 分制）且**步骤 ≥3 步**的成功任务轨迹。  
  - **进化步骤**：  
    1. **变异**：对 Skill 文件的关键段落生成多个改写版本。  
    2. **评估**：通过轻量模型（如 GPT-4o-mini）对版本打分。  
    3. **选择**：仅采纳**性能提升 ≥5%** 的版本更新原文件。  
  - **数据安全**：所有轨迹**本地存储**，默认不上传云端，需手动导出。  

### 2. **Atropos RL 训练管道标准**  
- **作用**：规范**从使用数据到模型微调**的闭环流程，面向高级用户。  
- **核心规则**：  
  - **数据导出**：任务轨迹需经 **人工审核** 才能用于训练，**禁止直接喂入原始数据**。  
  - **训练约束**：  
    - 仅支持 **工具调用精度** 和 **多步规划** 两类微调目标。  
    - **模型权重更新非自动触发**，需用户主动执行 `hermes train` 命令。  
  - **兼容性**：生成的训练数据需符合 **DSPy 框架**输入格式，确保跨模型适用。  

---

## 重点总结
- **记忆系统**：`MEMORY.md`/`USER.md` 的**字符限制**和**内容边界**是核心规则，确保记忆**高价值、低冗余**。  
- **技能学习**：`agentskills.io` 标准强制 Skill 文件包含**可验证的失败模式与决策逻辑**，**非简单步骤记录**。  
- **自主进化**：GEPA 与 Atropos 的规则**明确区分自动化层级**——  
  - 普通用户仅接触 **Skill 自动更新**（无需干预）。  
  - 模型训练需**主动触发**，且轨迹必须经人工筛选。  
- **所有规则均写入官方文档**：  
  - 记忆系统规范见 [Hermes Docs: Memory](https://hermes-agent.nousresearch.com/docs/user-guide/features/memory)。  
  - Skill 标准定义于 [agentskills.io](https://agentskills.io) 开放协议。  
  - GEPA 算法细节在 [hermes-agent-self-evolution](https://github.com/NousResearch/hermes-agent-self-evolution) 仓库中说明。  

这些文档共同构成 Hermes 的**可解释性基础**：其“自我进化”并非黑箱，而是**严格遵循预设规则的工程化流程**，避免行为失控或逻辑偏移。

Hermes Agent 是目前少数能做到“真的记住你，并且悄悄在变强”的开源 AI Agent。它打破了传统 AI “每次对话都从零开始”的局限，其核心在于一个“**执行→沉淀→复用→进化**”的持续闭环机制。我们不妨用一个具体例子来模拟这个过程，以便理解：

> **场景**：你好不容易让 AI 成功完成了一个多步骤操作，比如“结合本地日志，排查线上服务故障”。

*   **❌ 传统 AI**：任务结束，经验归零。下次遇到类似问题，一切都要重头再来。
*   **✅ Hermes Agent**：任务结束后，系统会自动复盘，将整个排查流程（包括成功路径和踩过的坑）打包成一个全新的**技能**存入库中。下次再遇到类似故障，它可以瞬间调用，无须重新指导。

### 🧠 记忆系统：从“金鱼”到“大象”

Hermes 并非提供单一的简单存储，而是构建了精准高效的多元化记忆架构：

*   **🎯 核心特质记忆层 (L1)**：**核心身份与风格**。通过两个严格限容、自动更新的纯文本文件，实现“越记越精炼”，而非“越记越臃肿”。
    *   **`MEMORY.md` (知识库)**：约 800 Token，存储环境知识、项目约定等。
    *   **`USER.md` (用户画像)**：约 500 Token，记录沟通偏好、常用工具链等。
    *   > “**我想分析上个月销售数据的趋势**”：启动时会注入，理解专业术语和分析偏好。
*   **🌊 情景记忆库 (L2)**：**过去聊了什么**。基于 `SQLite + FTS5` 的跨会话全文检索引擎。
    *   > “**我上次让你分析数据时，用了哪个脚本？**”：通过关键词检索，定位具体历史对话。
*   **📈 融合记忆库 (L3)**：**更复杂的关联信息**。采用更灵活的结构化记忆方案，支持更复杂的知识图谱与关系关联。
    *   > 当记录“那个分析脚本是为 CFO 准备的报告”时，能将“CFO”与“报告”等概念或事件关联起来。

### 📦 技能自主学习：从“程序员喂食”到“自己觅食”

Hermes Agent 的“自学习”能力，意味着它能像人类一样从过往经验中自动提炼方法论：

*   **自动生成 (Auto-Generation)**：完成任务后，若达到预定标准（如工具调用超 5 次、从错误中恢复），它将自动把“思考与操作”流程固化为标准技能卡，无需人工“写代码”。
*   **自我修复 (Self-Healing)**：主动检测技能是否失效。当某技能因外部环境或 API 变更失效时，能自动执行“补丁”精准修复。
*   **40+ 基础工具箱**：内置覆盖文件、网页、终端等能力的丰富工具，构建了广泛可攀爬的“技能树”。

---

### 🚀 自主进化：从“执行命令”到“规划未来”

“自主进化”是更高维度的“学习”，是整个智能体系在持续使用中不断自我完善的表现：

*   **自进化运行机制**：Hermes 将工作记忆、推理与反思逻辑嵌入循环，用 `Nudge Engine` 像闹钟一样提醒它定期复盘和整理经验。
*   **闭环数据飞轮**：不只生成技能，还会完整保存任务执行轨迹。这些高质量数据可用于反向微调大模型，实现“越用越强”。
*   **自进化模块架构**：其 `SkillEngine` 就像一个“方法论提炼机”，能自主将执行轨迹解析为可复用的知识图谱，存储在向量库中，形成专属的“经验财富”。

---

### ✨ 总结：一个会“成长”的助手

总的来说，Hermes Agent 为我们展示了一个前所未有的AI发展路径：一种从被动执行任务的工具，进化为能主动学习、积累经验和持续成长的智能伙伴。
