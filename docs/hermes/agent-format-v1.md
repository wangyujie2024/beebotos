

Hermes Agent 项目中定义了**三类核心规则标准文档**，主要用于规范 Agent 的行为逻辑、项目约束和长期记忆。其中 **`AGENTS.md`（项目级规则）和 `SOUL.md`（全局人格）是必须掌握的核心标准**，其余为兼容性或辅助性文档。以下是具体分类说明：

---

## 一、核心规则文档
### 1. `AGENTS.md`：项目级行为规范  
- **作用**：定义**当前项目的技术栈、编码规范、禁止事项及工作流程**，确保 Agent 自动遵守项目约定，避免低级错误（如忽略虚拟环境、修改错误文件等）。  
- **位置**：项目根目录（支持递归向上查找至 Git 仓库根目录）。  
- **关键内容**：  
  - **技术栈要求**（如 FastAPI 后端需用 `async/await`、测试目录为 `tests/`）。  
  - **禁止操作**（如禁止提交 `.env` 文件、禁止直接修改 `main` 分支）。  
  - **流程规范**（如 API 变更后需自动生成 TS 类型文件）。  
- **最佳实践**：  
  - 保持**简洁**（避免过长导致 Token 消耗增加），仅写入**高频复用的硬性规则**。  
  - 与 `CLAUDE.md`、`.cursorrules` **兼容**，若已使用 Cursor 编辑器，可直接复用现有规则文件。  

### 2. `SOUL.md`：全局人格与沟通规范  
- **作用**：定义 Agent **跨所有项目的长期行为风格**（如简洁/详细、技术化程度），而非单次任务指令。  
- **位置**：`~/.hermes/SOUL.md`（全局配置，跟随用户而非项目）。  
- **关键内容**：  
  - **沟通语气**（如“回答需简洁，能用一句话不用三句”）。  
  - **执行偏好**（如“修改文件前先读取内容”“复杂任务需分步确认”）。  
  - **不确定时的处理方式**（如“避免猜测，直接声明不确定性”）。  
- **最佳实践**：  
  - 仅写入**长期稳定的行为准则**，避免放入临时工作流或项目特定规则。  
  - 可通过内置预设快速切换风格（如 `/personality technical` 切换为技术专家模式）。  

---

## 二、兼容性规则文档
### 1. `.cursorrules` 与 `.cursor/rules/*.mdc`  
- **作用**：**直接兼容 Cursor 编辑器的规则文件**，若项目已配置 Cursor 规则，Hermes 会自动读取，无需重复定义。  
- **位置**：项目根目录或 `.cursor/` 子目录。  
- **适用场景**：从 Cursor 迁移至 Hermes 时，**无需修改现有规则**即可复用。  

### 2. `CLAUDE.md`  
- **作用**：与 **Anthropic Claude Code 兼容的项目规则**，功能与 `AGENTS.md` 类似。  
- **优先级**：Hermes 在加载项目上下文时，会按 `HERMES.md` → `AGENTS.md` → `CLAUDE.md` → `.cursorrules` 顺序**优先匹配首个存在文件**，**不会叠加**。  

---

## 三、辅助性文档（非规则标准，但影响行为）
### 1. `MEMORY.md` 与 `USER.md`  
- **作用**：  
  - `MEMORY.md`：存储**环境事实与经验教训**（如“CI 使用 GitHub Actions”）。  
  - `USER.md`：记录**用户偏好**（如“回答需简洁”“默认用 pytest”）。  
- **与规则文档的区别**：  
  - 属于**记忆系统**而非规则标准，用于补充上下文，**不定义强制行为**。  
  - 受字符限制（`MEMORY.md` 约 2,200 字符，`USER.md` 约 1,375 字符），需**精选高价值信息**。  

---

## 重点总结
- **核心规则必须写入 `AGENTS.md` 和 `SOUL.md`**：前者约束**项目内行为**，后者定义**全局人格**。  
- **避免冗余**：规则文档需**精简聚焦**，过长的 `AGENTS.md` 会显著增加 Token 消耗并稀释关键信息。  
- **兼容性优势**：Hermes **原生支持 Cursor/Claude 规则文件**，迁移成本极低，无需重复配置。  
- **与记忆系统的分工**：  
  - 规则文档（`AGENTS.md`/`SOUL.md`）定义 **"必须遵守的硬性约束"**。  
  - 记忆文件（`MEMORY.md`/`USER.md`）存储 **"可选参考的事实偏好"**。  

若需快速上手，建议优先配置 `AGENTS.md`（项目级）和 `SOUL.md`（全局级），这两者是 Hermes 实现 **"开箱即用且越用越准"** 的核心机制。
